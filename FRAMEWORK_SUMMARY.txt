KEY ARCHITECTURAL DECISIONS
================================================================================

1. PROJECTION MAPPING FORMAT
   - Unique ID: "{block_id}_{proj_name}"
   - Example: "block_0_q_proj"
   - Standardized across all modules

2. MAPPING REPRESENTATION
   - Dictionary: {proj_id: tile_id}
   - Enables efficient updates and change tracking

3. MAPPER INTERFACE
   - All mappers inherit from BaseMapper
   - Consistent API: compute_mapping(), apply_mapping(), get_stats()
   - Extensible for new strategies

4. ADAPTIVE MAPPER ALGORITHM
   - Greedy assignment: Sort projections by sensitivity, tiles by fidelity
   - Migration-aware: Cost = quality_improvement vs. weight_moved
   - Decision logic: Remap only if improvement > max(threshold, migration_cost)
   - Hysteresis: Cooldown period prevents oscillation

5. TILE FIDELITY MODEL
   - Heterogeneous initialization: High (1.0), Medium (0.7), Low (0.4)
   - Three degradation scenarios: gradual drift, localized fault, thermal
   - Per-timestep updates enable realistic degradation simulation

================================================================================
CORE ALGORITHM (AdaptiveMapper)
================================================================================

Objective:
  min_{M_t} Σ S_i * D_{M_t(i)}(t) + λ * Σ B_i * 1[M_t(i) != M_{t-1}(i)]

Components:
  - S_i: Projection sensitivity (normalized)
  - D_j(t): Tile degradation/fidelity at time t
  - M_t(i): Tile assigned to projection i at time t
  - B_i: Projection size in bytes
  - λ: Trade-off parameter (migration_threshold)

Decision Logic (per timestep):
  1. Compute current error: E_current = Σ(sensitivity × (1 - tile_fidelity))
  2. Generate candidate mapping: greedy assignment
  3. Compute candidate error: E_candidate
  4. Estimate improvement: improvement = (E_current - E_candidate) / E_current
  5. Estimate migration cost: cost = weight_moved / total_weight
  6. Remap if: improvement > max(threshold, cost) AND cooldown_elapsed

RESEARCH PHASES
================================================================================

Phase 1: PROJECTION-SENSITIVITY PROFILING
  Goal: Measure noise sensitivity of each projection
  Input: GPT-2 model, dataset, noise parameters
  Output: Sensitivity scores, perplexity metrics
  Duration: ~2-3 weeks
  
  Key Steps:
    - Load model and dataset
    - For each projection: inject noise at multiple levels
    - Measure perplexity and KL-divergence
    - Compute normalized sensitivity scores
    - Save profiles to data/profiles/

Phase 2: HETEROGENEOUS TILE-FIDELITY MODEL
  Goal: Validate time-varying tile fidelity simulation
  Input: Hardware config, degradation scenario
  Output: Fidelity evolution over timesteps
  Duration: ~1 week
  
  Key Steps:
    - Initialize 64 tiles with 3 fidelity classes
    - Simulate degradation (gradual/localized/thermal)
    - Validate fidelity tracking
    - Produce distribution plots

Phase 3: STATIC MAPPING BASELINES
  Goal: Establish baseline mapping strategies
  Input: Projections, tiles, sensitivities
  Output: Baseline evaluation results
  Duration: ~1 week
  
  Baselines:
    - Random: random tile assignment
    - Sequential: round-robin assignment
    - Hardware-only: assign by tile fidelity only
    - Static-sensitive: greedy by sensitivity (no adaptation)

Phase 4: ADAPTIVE MAPPING ALGORITHM
  Goal: Develop migration-aware dynamic mapping
  Input: All Phase 1-3 components
  Output: Trade-off curves, remapping analysis
  Duration: ~2 weeks
  
  Evaluate:
    - Adaptive mapper with varying thresholds
    - Comparison with non-cost-aware adaptation
    - Trade-off: quality vs. remapping overhead

Phase 5: COMPREHENSIVE EVALUATION
  Goal: Compare all strategies, generate insights
  Input: Results from Phases 1-4
  Output: Comparison tables, figures, recommendations
  Duration: ~2-3 weeks
  
  Metrics:
    - Language-model perplexity
    - KL divergence from clean outputs
    - Sensitivity-weighted tile error
    - Remapping events and weight volume
    - Inference latency and energy
    - Combined overhead
